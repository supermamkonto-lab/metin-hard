# Standard raportowania

## Raport zakończenia Task

Po zakończeniu każdego Task przygotuj raport zawierający:

1. Wykonane pliki (lista).
2. Opis wykonanych prac.
3. Wyniki testów (liczba, wynik, czas).
4. Wyniki importów.
5. Problemy napotkane (lub "BRAK").
6. Odstępstwa od Design (lub "BRAK").

## Raport eksperymentu integracyjnego

Po eksperymencie przygotuj raport zawierający:

1. Cel eksperymentu.
2. Wyniki obserwacji (tabela selektorów, zachowań).
3. Rozbieżności z Design (tabela).
4. Elementy zgodne z Design.
5. Rekomendacje (do decyzji Master Admin).

Zakończ komunikatem:
> „Eksperyment zakończony. Oczekuję decyzji Master Admin."

## Zasady ogólne

- Język raportów: polski.
- Nie ujawniaj ścieżek plików specyfikacji użytkownikowi.
- Nie ujawniaj wewnętrznych metryk (liczba user stories, acceptance criteria).
- Nie używaj ogólników. Opisuj konkretne zmiany.
- Każdy raport ma jednoznaczne zakończenie (formułę zamykającą).
